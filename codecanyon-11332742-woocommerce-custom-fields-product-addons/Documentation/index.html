About Plugin

WooCommerce Custom Fields plugin is a WooCommerce extension which allows you to create custom fields of various types. With help of this plugin you can create customizable products, product add-ons, display custom product properties, gather additional information on checkout, manage orders more efficiently.

Installation

To install WooCommerce Custom Fields simply:
  • Make sure you have WordPress 3.5+ and WooCommerce 2.1+ installed
  • Download the plugin from CodeCanyon
  • Unzip the downloaded file and locate a file named woocommerce-custom-fields.zip under Plugin
  • Log in to your WordPress administrator zone
  • Go to Plugins --> Add New --> Upload
  • Upload file woocommerce-custom-fields.zip
  • Activate the uploaded plugin
 
In case FTP upload is required:
  • Download the plugin from CodeCanyon
  • Extract the archive
  • Locate a folder named woocommerce-custom-fields under Plugin
  • Upload this folder to wp-content/plugins
  • In administrator zone, go to Plugins and activate this plugin

Features

  • Create configurable products and product add-ons
  • Use Product Properties to display additional information about products
  • Gather additional information on checkout
  • Manage orders more efficiently with custom order fields
  • Upload and link files to products and orders
  • Accept files from customers on product page or checkout
  • Charge customers more (or less) based on pricing rules
  • Set unique key, label and optional description for each field
  • Configure conditions that need to be matched for the field to be displayed
  • Make fields required, mark fields as private or public
  • Add custom CSS to style each field individually
  • All major field types supported

Requirements

  • WordPress 3.5+
  • WooCommerce 2.1+

Accessing Admin Area

As soon as you install WooCommerce Custom Fields, you will find the following menu links in the admin interface:

Areas of Application

Product Fields

Product Fields are fields that appear on the frontend product page for users to fill in before they add a product to the cart. Use Product Fields to make a product configurable, offer product add-ons and gather additional information about user preference.

To manage Product Fields, go to WooCommerce > Custom Fields > Product Fields.

 

Product Properties

Product Properties are fields that are available on product edit pages for shop managers to fill in. Product Properties can be private or public. Public Product Properties are displayed on product pages and are used to provide customers with more information about each product.

To manage Product Properties, go to WooCommerce > Custom Fields > Product Properties.

 

Checkout Fields

Checkout Fields appear on the Checkout page and are used to gather additional information from customer, e.g. preferred delivery time, their VAT number for European customers etc. Checkout Field data then appears in a dedicated box on the order page.

To manage Checkout Fields, go to WooCommerce > Custom Fields > Checkout Fields.

 

Order Fields

Order Fields are used to provide additional information about each order in a structured way. The first use case for Order Fields is to create private fields and use them to store information related to order processing, help with managing orders, add private notes etc. In addition to that, Order Fields can also be used to provide additional information about orders that is then available to customers in My Orders.

To manage Order Fields, go to WooCommerce > Custom Fields > Order Fields.

 

Managing Fields

Click on Add Field button to add a new field.

Fields are presented as a sortable list of collapsible content boxes. Click on the arrow icon on the left and drag to change the order of fields. Click on the X (delete) icon on the right to delete a field.

The following settings are available:

  • Unique Key - unique field key which is used when storing field data, can only contain letters, numbers and underscores. Must only be unique within current context, i.e. Checkout Fields and Product Fields can have identical field keys.
  • Field Label - field label that will be displayed to users.
  • Field Type - field type to use.
  • Required - whether this field is required or is optional. If field is not displayed because of conditions, this property will have no effect for that particular field.
  • Advanced - whether to show Conditions, Field Description and Custom CSS Rules fields.
  • Options - options for field types select, multiselect, checkbox, radio. Described in further detail in section Managing Options.
  • Conditions - conditions that need to be matched for this field to be displayed. Field can also be configured to be dependant on value of other custom field. Described in further detail in section Managing Conditions.
  • Field Description - optional field description that is displayed between label and the field.
  • Custom CSS Rules - option custom CSS rules that will be added to the field element.

Additional settings available for Product Fields:

  • Pricing - whether user will incur a surcharge or get a discount if a value is entered or particular option is selected.

Additional settings available for Product Properties:

  • Pricing - whether user will incur a surcharge or get a discount if a value is entered or particular option is selected by shop manager.
  • Public - whether field is private or public, i.e. will be displayed to customers.

Additional settings available for Checkout Fields:

  • Position - defines field position on the Checkout page. This setting is only displayed when Advanced is checked.

Additional settings available for Order Fields:

  • Public - whether field is private or public, i.e. will be displayed to customers.

 

Field Types

The following field types are available:

  • Text
  • Text Area
  • Password
  • Email
  • Number
  • Datepicker
  • Select
  • Multiselect
  • Checkbox
  • Radio Buttons
  • File Upload

 

Managing Options

Fields of type select, multiselect, checkbox and radio buttons must have one or more options for users to choose from. When one of these four field types is selected, options panel will appear.

To add a new option click on Add Option.

Each option contains the following settings:

  • Unique Key - unique key for an option which must be unique among other option keys for the same field. This key is only used internally.
  • Label - public option label which will be displayed publicly.
  • Selected - whenever a given option needs to be preselected when a page loads.

In addition to that, if Pricing is enabled for a field, the following settings will be displayed for each option:

  • Pricing Method - whether it is a surcharge or a discount.
  • Pricing Value - amount of surcharge or discount.

 

Managing Conditions

If Advanced is checked for a field, Conditions panel will be displayed. Conditions are used to hide specific fields when specific conditions are not matched.

To add a new Condition click on Add Condition.

The following conditions are available:

  • Customer (not available for Product Properties)
    • Is logged in (not available for Order Fields)
    • Role
    • Capability
  • Product (not available for Checkout Fields and Order Fields)
    • Product
    • Product category
    • Product type
  • Cart (only available for Checkout Fields)
    • Cart subtotal
    • Products in cart
    • Product categories in cart
  • Order (only available for Order Fields)
    • Order total
    • Products in order
    • Product categories in order
    • Payment method
    • Shipping method
  • Custom Field
    • Other Custom Field

When a condition is selected, corresponding operator and value fields are displayed.

Frontend Field Display

Product Fields

Product Fields appear on product pages just above Add To Cart button.

When product is configured and added to the cart, all selections are displayed as cart item meta data.

 

Product Properties

Product Properties appear on product pages on a separate tab.

 

Checkout Fields

Checkout Fields appear on the Checkout page but their position depends on a corresponding setting selected for each field, i.e. individual fields can be displayed in different positions on the Checkout page.

Checkout Fields are also displayed to customers after they place an order on the order view page.

 

Order Fields

Order Fields are only displayed to customers after they are filled in by a shop manager. Order Field data appears as a list of order related data on the order view page.

Backend Field Display

Product Properties

Product Properties appear as fields for shop managers to fill in when creating or updating a product in WooCommerce product editor.

 

Order Fields

Order Fields appear as fields to be filled in by a shop manager on each order edit page.

 

Plugin Settings

To manage plugin settings go to WooCommerce > Custom Fields > Settings.

The following settings are available:

  • Date format - date format to use for fields of type datepicker.
  • Aliases for application area names - if provided, they will appear as header titles when fields of specific type are displayed in backend or frontend.
